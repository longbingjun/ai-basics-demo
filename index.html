How to Use AI · 基础认知篇

业务人员如何理解 AI

精简版入门培训：4 个必知概念 → 翻译例子串讲 → 行动原则与进阶路径。约 16 页，适合 45–60 分钟场次。

培训目标：会用 Prompt · 理解模型在「猜」不是「查」· 知道必须人工复核

Step 1 · 你写了什么（Prompt）

用户希望 AI 把一句中文翻译成自然英文，并且不要添加额外含义。

Prompt
请把「我想要学习英语」翻译成自然英文，不要添加额外含义。
期望输出
I want to learn English.

任务说明书：做什么（翻译）、翻译什么、约束是什么。

Step 2 · 模型先「读」

Prompt 被切成 token 后，模型先读完再开始写。中文部分大致切分为：

我想要学习英语

连同你的 Prompt 和历史对话，一起构成模型这一轮的「上下文」。

Step 3 · 模型再「写」（逐 token 概率生成）

模型在 Prompt 后面，一个 token 一个 token 地预测输出：

已生成 下一个 token 为什么概率高
（空）I翻译句常以 I 开头
IwantI 后接 want 频率极高
I wanttowant 后常接 to
I want tolearn「学习」→ learn
I want to learnEnglish「英语」→ English
I want to learn English.句子结束

不是查词典，而是逐步选概率最高的 token——即第一部分的「概率生成」。

Step 4 · 能力从哪来？为什么有多种译法？

预训练：模型在海量文本上练习「猜下一个 token」，学会了中英文之间的统计规律。后训练：进一步学会听 Prompt 指令（如「请翻译」「不要添加额外含义」）。

为什么有多种合理英文？每一步都在多个候选中选概率较高的，不是查唯一标准答案——所以需要人工复核，也可在 Prompt 里约束风格。

1. 说清目标明确你要 AI 做什么，而不是只说「帮我看看」。
2. 给足上下文提供规则、文件、样例和限制条件；内容太长时，最早的要求可能被挤掉。
3. 写明边界说明哪些不要擅自判断，哪些必须人工确认。
4. 先计划再执行复杂任务先让 AI 说明步骤，再授权它动手。
5. 复核结果涉及事实、数字、合规的内容，必须有人核实——模型不会自行检验对错。
6. 需要依据时把相关资料放进 Prompt，或使用 RAG（检索增强生成：先从文档库找段落，再让模型回答），可降低瞎编风险。

任务变复杂、变重复时，按下面三级逐步演进——不是一开始就要全用上。

1

聊天式使用

对话框里问一句、答一句，靠 Prompt 临时说明需求。适合探索性、一次性任务。

2

文档化（Documents）

把规则、术语、输出格式写进固定文档（如 .md / .txt），给人和 AI 共同阅读，不用每次在聊天里重复粘贴。

3

流程化（Workflow）

多步骤、常重复的任务，把顺序和规则固化成固定流程或 checklist，减少每次重新描述。

选读 · 知道名字即可
Reference（参考依据）、Script（脚本）、Skill（技能包）、Agent（智能体）、Git（版本管理）——属于更深入的工程化能力，入门培训不必展开。当你已经稳定使用文档和流程后，再按需了解。
