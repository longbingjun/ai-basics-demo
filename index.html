How to Use AI · 基础认知篇

用更像培训课件的方式理解 AI

这版不把所有内容堆在一页里，而是按章节点击切换。先讲基础概念，再讲模型链条，再用翻译例子串起来，最后过渡到项目化使用。

Token

AI 读写文字时使用的小颗粒，不一定等于一个字或一个词。

Prompt

用户给 AI 的任务说明书，告诉它要做什么、怎么做、不要做什么。

上下文

AI 当前能看到的信息，包括问题、文件、规则、样例和历史对话。

推理

模型在使用时，根据输入和上下文生成答案的过程。

幻觉

AI 生成了看似合理但没有依据或不正确的内容。

RAG / 工具调用

让 AI 基于指定资料回答，或调用工具读文件、写代码、运行命令。

数据收集模型先接触大量文本、代码、网页、书籍、问答和翻译样例。
预训练模型从海量资料中学习语言规律、知识关系和表达模式。
后训练模型进一步学习如何按人的指令回答、遵守格式和边界。
用户输入用户通过 Prompt 把任务、规则、语气和输出要求交给模型。
推理生成模型结合上下文，一步步生成最可能合适的内容。
人工复核人检查事实、业务口径、金额、合规和最终判断是否可靠。

一句翻译任务

用户希望 AI 把一句中文翻译成自然英文，并且不要添加额外含义。

用户输入
请把“我想要学习英语”翻译成自然英文，不要添加额外含义。
理想输出
I want to learn English.

这个例子足够简单，但能讲清楚 Token、Prompt、上下文、预训练、后训练、推理生成和人工复核。

Token 和 Prompt

Prompt 会被拆成 token，模型再基于这些小颗粒和上下文理解任务。

英文 I want to learn English. 可以粗略理解为：

IwanttolearnEnglish.

中文“我想要学习英语”可以粗略理解为：

我想要学习英语

Token 不需要精确计算，只要知道内容越长，占用上下文越多。Prompt 不是咒语，而是任务说明书。

预训练模型见过大量中英文文本，学到“我想要……”常对应 “I want to ...”，“学习英语”常对应 “learn English”。
后训练模型学会了看到“请翻译”就执行翻译任务，而不是随意续写；看到“不要添加额外含义”就避免加 only。
推理生成真正使用时，模型结合 Prompt、上下文和语言规律，生成 I want to learn English.

为什么不是唯一答案？

I want to learn English 自然通用；I would like to learn English 更礼貌；I want to study English 更偏课程学习；I only want to study English 多了“只想”的含义，不适合普通翻译。

能动手

Agent

大模型加上工具、权限和执行循环，能读文件、写脚本、跑任务、迭代修正。

能稳定

Script

把确定、重复、规则清楚的步骤写成可执行代码。

能重复

Workflow

一套固定做事流程，适合经常重复执行的任务。

有依据

Reference

执行任务时必须遵守或查询的标准依据，如字段定义、术语表、业务规则。

可复用

Skill

把触发条件、流程、规则、样例和工具打包成可复用能力。

可回退

Git

记录代码、规则和文档变化，方便协作、审查和回退。

1. 说清目标明确你要 AI 做什么，而不是只说“帮我看看”。
2. 给足上下文提供规则、文件、样例、术语表和限制条件。
3. 写明边界说明哪些不要擅自判断，哪些必须人工确认。
4. 先计划再执行复杂任务先让 AI 说明步骤，再授权它动手。
5. 复核并沉淀重要结果要检查，重复任务要变成 Workflow 或 Skill。
